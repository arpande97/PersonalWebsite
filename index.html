Work Samples

File Compression using Huffman Encoding

A file compression tool that using Huffman encoding/decoding, encodes your original file replacing fixed-length encoding of characters with variable-length encoding. Characters with higher frequency in the text get encoded with fewer bits than standard and bit-packed into a byte. As a sample test case, a 3.2MB file was compressed into a 2.1MB file. It was a good way to learn about the basic flavour of compression algorithms.

DNS Resolver

Understanding the DNS request/response packets and how authority name servers play a part in resolving the domain name was the motivation behind this project. It currently handles only 'A' type records and future enhancement will incorporate CNAME records as well. The easier way to do this would be to just use the InetAddress in Java, but working with individual data packets will enhance your knowledge about this frequently asked interview question.

4-approximation algorithm for a minimum zero skew tree

The objective was to implement an algorithm for finding zero- or bounded-skew trees of minimum total weight, since the weight of the tree is directly proportional to the amount of resources required in, for example, bandwidth and buffers for network multicasting, power and chip area for clock routing in VLSI. The skew of the tree is the maximum difference between any two root-to-leaf paths. A zero skew would then mean that the distance from any root-to-leaf path is similar, leading to "zero skew". This ensures that any clock signal from a source reaches all the nodes at exactly the same time.
